# 特关热点规则设计

日期：2026-06-26
状态：已实现

## 背景

AI HOT Notifier 当前会轮询 AI HOT API，将新内容写入 `history`，并通过普通通知提醒用户。用户希望对特定来源、作者或关键词的热点做特关，例如“公众号：数字生命卡兹克”的文章，以便及时收到更醒目的提醒，并能在弹窗中方便找回查看。

## 目标

- 支持配置多条特关规则。
- 规则可按来源、作者、关键词组合匹配热点。
- 命中特关的热点单独通知，并在列表中醒目标记。
- 避免通知轰炸：单轮最多发送 3 条特关通知，并对未查看条目按固定节奏重复提醒。
- 已查看条目不再重复通知。
- 保持现有普通通知、已读状态、历史列表和旧存储数据兼容。

## 非目标

- 不新增外部依赖或构建流程。
- 不做复杂规则表达式、正则编辑器或独立管理页面。
- 不要求保存 AI HOT API 的全部原始字段；第一版继续基于 `source`、`title`、`summary`、`url` 等现有字段工作。
- 不改变扩展权限范围。

## 用户交互

### 设置面板

在现有设置面板中新增“特关”区域。

每条规则包含：

- 来源：可空，例如 `公众号`、`X`、`IT之家`。
- 作者：可空，例如 `数字生命卡兹克`、`Khazix09186`、`Kling_ai`。
- 关键词：可空，多个关键词用中英文逗号分隔，例如 `Claude Code, Hook`、`Claude Code，Hook`。
- 启用状态：规则可启用或停用。
- 删除操作：可删除不再需要的规则。

新增规则时使用轻量表单，不跳转新页面。输入后添加为规则项；相同来源和作者的规则会合并关键词。规则项支持启停、删除整条规则，并支持删除单个关键词。

规则项排版：

- 首行展示来源、作者、启停按钮和删除按钮。
- 来源完整显示；作者占用剩余空间，过长时在操作按钮前省略。
- 关键词只在存在时另起一行展示，并横向扩展到规则项最右侧；未设置关键词时不预留空行。
- 添加新规则后，规则列表滚动到底部，露出刚添加的规则。

### 弹窗列表

- 未读的特关条目在主历史列表内置顶展示。
- 多条未读特关按发布时间倒序排列。
- 点击条目后标记为已读和已查看，并恢复到默认时间线位置。
- 条目底部保留“特关”标签，方便在完整时间线中识别。

## 匹配规则

### 字段来源

插件保存到 `history` 的字段包括：

- `source`：来源和作者文本，例如 `公众号：数字生命卡兹克`、`X：可灵 Kling AI (@Kling_ai)`。
- `title`：标题。
- `summary`：摘要。
- `category`：分类。
- `url`：原文链接。
- `time`：发布时间。
- `discoveredAt`：插件发现时间。

### 解析方式

- 来源匹配：从 `source` 的冒号前缀解析，例如 `公众号：数字生命卡兹克` 的来源为 `公众号`，`X：可灵 Kling AI (@Kling_ai)` 的来源为 `X`。
- 作者匹配：使用 `source` 中冒号后的文本匹配，包括中文名和括号内账号。例如 `数字生命卡兹克`、`Kling_ai` 都可作为作者关键词。
- 关键词匹配：匹配 `title` 与 `summary` 拼接后的文本。

### 逻辑

- 同一条规则内，填写的条件必须全部命中。
- 空条件不参与限制。
- 关键词支持多个值，以英文逗号或中文逗号分隔；多个关键词命中任意一个即可。
- 文本匹配采用大小写不敏感的包含匹配，并在匹配前去除首尾空格。
- 如果规则没有填写任何有效条件，则视为无效规则，不参与匹配。

示例：

| 来源 | 作者 | 关键词 | 命中含义 |
| --- | --- | --- | --- |
| `公众号` | `数字生命卡兹克` | 空 | 匹配数字生命卡兹克公众号的全部内容 |
| `公众号` | `数字生命卡兹克` | `Claude Code, Hook` | 匹配数字生命卡兹克公众号中标题或摘要含 Claude Code 或 Hook 的内容 |
| `X` | `Kling_ai` | 空 | 匹配 X 上可灵 Kling AI 账号的内容 |

## 通知策略

### 首次通知

每次轮询发现新内容后，先识别特关条目。

- 特关条目使用独立通知。
- 普通通知排除这些特关条目，避免同一内容重复通知。
- 单轮最多发送 3 条特关通知。
- 如果同一轮命中特关超过 3 条，优先通知最新 3 条，其余只进入历史列表。

### 重复提醒

未查看的特关条目按以下节奏提醒：

| 阶段 | 时间 |
| --- | --- |
| 首次提醒 | 立即 |
| 快速提醒 1 | 首次后 2 分钟仍未查看 |
| 快速提醒 2 | 首次后 5 分钟仍未查看 |
| 后续提醒 | 首次后 2 小时仍未查看 |
| 长期重复 | 之后每 24 小时最多一次 |

重复提醒也受“单轮最多 3 条特关通知”限制。

### 已查看判定

以下行为将特关条目标记为已查看：

- 点击特关通知并打开原文。
- 在弹窗中点击该条内容并打开原文。
- 点击“全部已读”。

仅打开插件弹窗但未点击条目，不算已查看。

已查看条目不再重复发送特关通知。

## 存储设计

### `watchRules`

保存特关规则列表。

```json
[
  {
    "id": "wr_...",
    "source": "公众号",
    "author": "数字生命卡兹克",
    "keywords": ["Claude Code", "Hook"],
    "enabled": true,
    "createdAt": "2026-06-26T00:00:00.000Z"
  }
]
```

### `watchNotifyState`

按 `url` 保存特关通知状态。

```json
{
  "https://mp.weixin.qq.com/s/example": {
    "ruleIds": ["wr_..."],
    "firstMatchedAt": "2026-06-26T02:11:27.378Z",
    "lastNotifiedAt": "2026-06-26T02:11:27.378Z",
    "notifyCount": 1,
    "nextNotifyAt": "2026-06-26T02:13:27.378Z",
    "viewedAt": ""
  }
}
```

字段含义：

- `ruleIds`：命中的规则 ID 列表。
- `firstMatchedAt`：第一次匹配特关的时间。
- `lastNotifiedAt`：最近一次特关通知时间。
- `notifyCount`：已发送特关通知次数。
- `nextNotifyAt`：下次允许提醒时间。
- `viewedAt`：查看时间；非空表示不再重复通知。

### `history` 扩展字段

命中特关的历史条目增加可选字段：

```json
{
  "watchMatched": true,
  "watchRuleIds": ["wr_..."],
  "watchMatchedAt": "2026-06-26T02:11:27.378Z"
}
```

旧历史条目没有这些字段时按普通条目处理。

## 后台流程

1. 轮询 API，得到新条目。
2. 读取 `watchRules` 和现有 `history`。
3. 对新条目执行特关匹配。
4. 将新条目转换为 history entry；命中特关时写入 `watchMatched` 等字段。
5. 写入更新后的 `history`。
6. 对需要提醒的特关条目发送通知，最多 3 条。
7. 对非特关的新条目执行现有普通通知逻辑。
8. 更新 badge。
9. 使用 alarm 或现有轮询入口检查到期的未查看重复提醒，确保 2 分钟、5 分钟提醒不依赖下一次 API 有新数据。

## 弹窗流程

1. 加载 `history`、`readIds`、`readAllBeforeByMode`、`watchRules`、`watchNotifyState`。
2. 渲染设置面板中的规则列表。
3. 从 `history` 中筛选未读的 `watchMatched` 条目，按发布时间倒序置顶到主列表。
4. 主历史列表不重复渲染已置顶条目；已读特关恢复默认时间线位置。
5. 命中特关的条目增加标签和强调样式。
6. 点击条目时，沿用现有已读逻辑，并同步设置对应 URL 的 `watchNotifyState.viewedAt`。
7. 点击“全部已读”时，同时将当前可见或历史窗口内的特关条目标记为已查看。

## 错误处理与兼容

- `watchRules` 缺失、为空或格式异常时，按无特关规则处理。
- 单条规则字段异常时跳过该规则，不影响其他规则。
- `watchNotifyState` 缺失或某 URL 状态缺失时，按首次匹配处理。
- 旧版本 `history` 没有特关字段时，仍正常显示。
- 规则保存失败时，设置区域显示失败反馈，不影响历史列表使用。
- 通知发送失败时，不应阻断 history 写入；下次检查仍可根据状态重试。

## 测试计划

- 规则匹配：来源、作者、关键词、组合条件、空条件、大小写、中文逗号和英文逗号。
- 通知策略：新命中立即通知、单轮最多 3 条、普通通知排除特关、已查看后不重复。
- 重复提醒：2 分钟、5 分钟、2 小时、24 小时节奏和 `nextNotifyAt` 计算。
- UI 渲染：未读特关置顶且不重复；已读后恢复默认位置；主列表重点标签保留。
- 已查看同步：点击通知、点击列表项、点击全部已读都会写入 `viewedAt`。
- 存储兼容：缺失 `watchRules`、`watchNotifyState`、旧 `history` 字段时不报错。
